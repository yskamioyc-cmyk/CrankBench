## 3. 各週の詳細プロセスとアクションプラン

### ■ 第1週 (Week 1): 土台作りと数学のコード化
- **プログラミング**: 
  - 前回のTauri環境をベースに、Vite + TypeScript + `@react-three/fiber` `@react-three/drei` を導入。
  - 基本設計書にある「ピストンの幾何学数式」をRustの純粋な関数として実装。
- **3D表現**: 
  - Blender等の素材は使わず、まずはR3Fの標準パーツ（`cylinderGeometry` や `boxGeometry`）を組み合わせて、簡易的な「ピストン・コンロッド・クランク」の形を画面に配置する。

### ■ 第2週 (Week 2): 【最重要】3Dデータバインディングの確立
- **プログラミング**: 
  - フロントのスライダー（ボア・ストローク）の値をTauri InvokeでRustに送り、Rust側で「形状比率」と「クランク角ごとの座標配列」を一瞬で計算して返すパイプラインを構築。
  - `requestAnimationFrame`（またはR3Fの `useFrame`）を利用し、クランクが毎秒60フレームで滑らかに回転するアニメーションを実装。
- **就活対策の視点**: ここが完成した時点で「動くデモ」として他者に見せられる状態（MVP）になるため、開発の最大のリスク（3Dが動かない）を回避できます。

### ■ 第3週 (Week 3): 熱力学のロジック肉付け
- **プログラミング**: 
  - 物理・数学の学習を並行。高校物理〜大学基礎レベルの「断熱変化の式（$PV^\kappa = C$）」をRustに実装。
  - 横軸にシリンダ容積、縦軸に圧力をとった「PV線図」を2Dグラフ（Recharts）で表示。3Dのピストンが一番上に上がったとき（上死点）に、圧力グラフのピンが最高点を指すように同期させる。
- **3D表現**:
  - 圧力・温度の数値に応じて、3Dシリンダの色（マテリアルカラー）をリアルタイムに変化させる。

### ■ 第4週 (Week 4): 「爆発」の実装と魅せる工夫
- **プログラミング**: 
  - 点火時期（BTDC）のスライダーを追加。Wiebe関数を組み込み、点火された瞬間に圧力が急上昇する「本物のエンジンらしい」PV線図を完成させる。
  - 計算された「仕事量」から、エンジンの回転数に応じた「図示馬力（PS）」「トルク（Nm）」を計算し、画面のインジケータに表示。
- **仕上げ（就活特化）**:
  - コードをきれいにリファクタリング。
  - GitHubのREADMEに、**「スライダーを動かすと3Dモデルがニュッと変形し、グラフの形が変わる様子」のGIFアニメ（または動画リンク）**を冒頭に貼り付ける（面接官が1秒で凄さを理解できるようにするため）。

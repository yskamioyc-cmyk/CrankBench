# 要件定義書：3D熱力学エンジンシミュレータ (3D Engine Thermodynamic Simulator)

## 1. 開発背景と目的
自動車のDIY整備やチューニング（特にHA36Sアルトワークス等の現代的な電子制御車両）において、エンジンの燃焼理論、熱効率、熱管理の動態を深く理解するためのデスクトップアプリケーション。
前作のテキストエディタ開発で培ったTauri＋React＋Rustの基盤をベースに、Web 3Dグラフィックス（WebGL/Three.js）技術を融合。ユーザーがUI（スライダー）でパラメータを動的に変更した際の影響を、**リアルタイムな数値計算（Rust）**と**インタラクティブな3Dモデル（React）**の双方で視覚化し、ポートフォリオとしての高い技術アピール度と内燃機関工学の学習効率を両立させることを目的とする。

## 2. 対象ユーザー
- 自動車の構造やセッティング理論（ECUマップ等）を学びたいプライベーター、DIY整備士
- フロントエンドにおける3D表現（WebGL）と、バックエンド（Rust）の高速計算の連携実績を評価する採用面接官

## 3. システム要件・ターゲット環境
- **プラットフォーム**: デスクトップアプリケーション（Windows / macOS）
- **技術スタック**:
  - フロントエンド: React, TypeScript, Vite, **React Three Fiber / @react-three/drei (3D描画)**, Recharts（グラフ描画）
  - バックエンド/計算エンジン: Rust (Tauri経由でのIPC通信)
  - デスクトップフレームワーク: Tauri v2
- **PCスペック**: 軽量動作を維持（WebGL対応の一般的なノートPC・内蔵グラフィックスで動作可能）

## 4. 機能要件 (MVP範囲および拡張計画)

### 4.1 パラメータ動的入力機能
ユーザーはフロントエンドのUI（スライダーおよびテキスト入力）を用いて、以下のパラメータをリアルタイムに変更できること。
1. **エンジン基礎諸元**:
   - ボア径 (mm) → **3Dモデルのピストン直径・シリンダ径にリアルタイム連動**
   - ストローク量 (mm) → **3Dモデルのクランク半径・ピストンの上下幅にリアルタイム連動**
   - コネクティングロッド長 (mm) → **3Dモデルのコンロッド長にリアルタイム連動**
   - 幾何学的圧縮比
2. **エンジン配置・レイアウト**:
   - 気筒数（1気筒、直列3気筒、直列4気筒、V型6気筒など）

### 4.2 計算エンジン機能 (Rustバックエンド)
1. **総排気量・単気筒容積計算**:
   - 入力された諸元から「単気筒容積」および「総排気量 (cc)」を即座に算出する。
2. **ピストン幾何学・座標計算 (フェーズ1)**:
   - クランク角 ($0^\circ \sim 720^\circ$) に対するシリンダ容積の変化に加え、**3Dレンダリング用のピストンY座標、コンロッドの傾き角度などを毎秒60フレーム（または任意のステップ）で超高速に計算してフロントエンドに渡す**。
3. **熱力学サイクル計算 (フェーズ2以降)**:
   - 理想気体の状態方程式に基づく筒内圧力・温度の計算。
   - 気筒数に応じた総出力（馬力・トルク）の合算。

### 4.3 ビジュアル・可視化機能 (フロントエンド)
1. **リアルタイム3Dビューア (新規要件)**:
   - 3D空間上にピストン、コネクティングロッド、クランクシャフトを配置し、アニメーション駆動させる。
   - スライダーでボアやストロークを変更した際、3Dモデルの形状（太さ・長さ）が**ニュッと動的に変形（リアルタイム・データバインディング）**すること。
   - 燃焼・爆発のタイミング（フェーズ3以降）で、シリンダ内部が視覚的に発光・変化するエフェクト。
2. **動的グラフ表示**:
   - 容積曲線グラフ（クランク角 vs 容積）やPV線図（圧力 vs 容積）のプロット。
   - 3Dモデルの動きとグラフのプロット位置が同期して動くこと。

## 5. スコープ外（やらないこと）
- 物理エンジンによるパーツ同士の「衝突判定」や「重力演算」（すべて幾何学数式による座標上書きで制御するため不要）
- 3D流体解析（CFDによる混合気の流れの可視化）

## 6. 開発マイルストーン (ロードマップ)

### フェーズ1：幾何学 ＆ 3D基礎（現在のターゲット）
- **ゴール**: スライダーでボア・ストロークを動かすと、画面上の簡易3Dパーツ（円柱などで表現）の太さや動きの幅がリアルタイムに変形し、排気量計算と連動する。
- **目的**: Tauriを介した「React(スライダー) ⇄ Rust(座標・物理計算) ⇄ React Three Fiber(3Dレンダリング)」の超高速なデータパイプラインを確立する。

### フェーズ2：熱力学・断熱圧縮ステージ
- **ゴール**: 燃焼のない圧縮状態を数式化。ピストンが上昇して空気が圧縮されるにつれ、3D空間上のシリンダの色が「青（冷）→ 赤（熱）」へと変化する視覚効果と、PV線図の描画。

### フェーズ3：点火・燃焼（Wiebe関数）ステージ
- **ゴール**: 点火時期パラメータを追加。爆発の瞬間に3Dシリンダが一瞬激しく発光（フラッシュ）するエフェクトを実装し、馬力・トルクの算出を可能にする。

### フェーズ4：マルチシリンダ ＆ 熱管理ステージ
- **ゴール**: 直3（アルトワークス仕様）やV6などの気筒数切り替えに対応し、3Dモデルを複数気筒化。連続運転による水温・油温の上昇シミュレーションを実装。

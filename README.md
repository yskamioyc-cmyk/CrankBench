# CrankBench 
> **2D Kinematic Engine Bench Simulator**

Rust（Tauri）の高速な計算性能と、React/TypeScriptによるモダンなUIを組み合わせた、リアルタイム熱力学・運動学エンジンシミュレータです。クランク機構の2Dアニメーションと、P-V線図・T-S線図の動的なプロットを、軽量かつ滑らかに実行します。また、生成されたデータから近似的にエンジンの性能曲線を算出し、簡易的な馬力（PS）・トルク（kgf・m）-RPM図を描画します。

---

##  主な機能

- **動的メカニズムシミュレーション**: ボア径、ストローク、コンロッド長、圧縮比をリアルタイムに変更し、エンジンの挙動を即座に再計算。
- **マルチレイアウト対応**: 単気筒だけでなく、直列3気筒（120°）、直列4気筒（180°）、直列6気筒（120°）のクランク位相変化を正確に再現。
- **熱力学サイクルプロット**: 
  - **P-V線図 (Pressure - Volume)**: 1気筒あたりの図示仕事を可視化。
  - **T-S線図 (Temperature - Entropy)**: 断熱過程や等容燃焼における熱力学的エネルギー推移を可視化。
- **性能予測（ベンチマーク）**: 生成されたデータから、最大9000 RPMまでのトルク曲線・出力曲線を内部でエミュレートし、最大トルク（kgf・m）および最高出力（PS）をリアルタイム算出。

##  技術スタック

- **Frontend**: React, TypeScript, Vite, Recharts
- **Backend (Desktop Shell)**: Rust, Tauri v2
- **Graphics**: HTML5 Canvas (2D Context)

##  描画最適化のアーキテクチャ

本プロジェクトでは、高頻度なデータ更新と滑らかな2Dアニメーションを両立するため、**「Reactのレンダリングサイクルからの脱却」**をテーマにしたハイブリッド設計を採用しています。

```
[Rust Backend] (高速計算)
       │  (Tauri IPC)
       ▼
[React App Component] ──(静的背景データ)──> [Recharts (グラフ)] 
       │                                         ▲
       │ (useRefによる制御)                      │ (15FPS 間引き更新)
       ▼                                         │
[Engine2D Component] ───(requestAnimationFrame)─┘
       │
       ├─(60FPS)─> [HTML5 Canvas] (ピストン・クランクの描画)
       └─(60FPS)─> [DOM Direct Update] (圧力・温度等のテキスト数値を直接書き換え)
```

1. **DOM Direct Update (60FPS)**:
   毎フレーム変化するクランク角や筒内圧力・温度などの数値テキストは、Reactの `State` を介さず、`useRef` を用いてブラウザのDOMへ直接インジェクションしています。これにより、Reactの不要な再レンダリング（仮想DOMの再計算）を100%抑制しています。
2. **描画負荷の分散**:
   メインのアニメーションおよびCanvas描画はブラウザの最高リフレッシュレート（60FPS+）で駆動させつつ、描画コストの高いグラフ（Recharts）の現在点インジケーターの更新のみを **15FPS** に制限（デバウンス）することで、メインスレッドの占有を防ぎ、カクつき（Jank）を完全に解消しました。
3. **データの間引き (Data Downsampling)**:
   グラフの背景線となる720点の膨大なシミュレーションデータを、見た目の滑らかさを損なわない180点に間引いてレンダリングすることで、SVG of Rechartsの描画負荷を大幅に削減しています。

##  セットアップと実行方法

### 前提条件
- [Node.js](https://nodejs.org/) (v18以上推奨)
- [Rust](https://www.rust-lang.org/) (cargo, rustc)
- 各OSに応じたTauriのシステム依存関係（Tauri公式サイトを参照）

### 1. リポジトリのクローン
```bash
git clone [https://github.com/YOUR_USERNAME/CrankBench.git](https://github.com/YOUR_USERNAME/CrankBench.git)
cd CrankBench
```

### 2. 依存関係のインストール
```bash
npm install
```

### 3. 開発モードでの起動（Tauri環境）
```bash
npm run tauri dev
```

### 4. 物理定数の変更
- 燃焼モデルやサイクルの変更を行う場合、物理定数を`./src-tauri/physics_model.toml`で分割管理しているので、その内容を変更してください。

## 熱力学的計算について
- このシミュレータでは、エンジンの燃焼効率計算にかかるそれぞれの数値を個別のエンジンごとに定めるのではなく、自動車用エンジンの中央値・平均値から算出し、目安値として定め使用しています。そのため、実在のエンジンの諸元を入力しても、カタログスペックの値とは乖離が発生します。
- 主な数値と、その選定理由は以下の通りです。

### 1. 燃焼圧力倍率：4.2
- 近年のエンジンでは高タンブル流の採用による急速燃焼により、燃焼圧力倍率は**4.5~5.2**倍近くまで上昇します。
- 年式が古いエンジンでは、ボア径が大きく、タンブル流（縦渦）ではなくスワール流（横渦）主体の設計のため、火炎伝播速度が比較的遅く、燃焼圧力倍率は**3.8~4.0**倍程度にとどまります。
- 中央値として**4.2**を採用することで、最新のエンジンほどの効率はないものの、古典的な設計のエンジンよりは燃焼効率がよい、おおよそ1990年代後半~2000年代の一般的な乗用車用エンジンの性能を計算により算出します。

### 2. 体積倍率: ピーク0.88
- 近年の実用エンジンではVVT(可変バルブタイミング機構)が装備されているため、低回転域から高回転域まで広い範囲で**0.90~0.95**という高い吸気効率が維持されます。
- 年式が古いエンジンではVVTの採用例が少ないため、過給エンジンの場合パワーバンドでは吸気効率が上昇しますが、バンドを外れると効率が低下します。一般的なVVTなしの実用エンジンの場合、吸気効率のピークは**0.85**前後です。

### 3. 機械損失: 0.10 + 0.000020 * n
- 近年の軽自動車エンジンなどは、フリクション低減を主眼に設計が行われており、高回転域でも**15~18%**の損失にとどまります。
- 20年ほど前のスポーツカー用直列6気筒エンジンなどでは、高出力に耐えるためクランクシャフトの軸径が太く、ピストンも重いなど、摩擦や機械抵抗の値が大きいです。高回転域では**25~30%**のエネルギーがフリクションロスとして消費されます。
- 掲載の数式で計算を行うと、レイアウトや排気量にかかわらず、「教科書的・平均的な」抵抗による損失を算出でき、中央値的なエンジン出力をシミュレートするのに有効です。
